# Инструкция для ручного тестирования API

**Дата:** 10 марта 2026  
**Приложение:** Selectel Vacancies API  
**Swagger UI:** http://localhost:8000/docs

---

## Эндпоинты для тестирования

### 1. GET /api/v1/vacancies/ — список вакансий

**Параметры query (опционально):**
- `timetable_mode_name` — фильтр по режиму работы
- `city` — фильтр по городу

**Пример запроса:**
```
GET http://localhost:8000/api/v1/vacancies/ ✅
GET http://localhost:8000/api/v1/vacancies/?city=Москва ✅
GET http://localhost:8000/api/v1/vacancies/?timetable_mode_name=Full ✅
GET http://localhost:8000/api/v1/vacancies/?city=москва  # lowercase ✅
GET http://localhost:8000/api/v1/vacancies/?city=Санкт   # частичное совпадение ✅
```

**Ожидаемый результат:**
- Статус `200 OK` ✅
- JSON-массив с вакансиями (должно быть ~28 вакансий после парсинга) ✅
- У каждой вакансии: `id`, `title`, `city_name`, `published_at`, и т.д. ✅
- Фильтр работает регистронезависимо (ilike) ✅

**Что проверить:**
- [x] Пустой запрос возвращает все вакансии
- [x] Фильтр по городу работает (регистронезависимо)
- [x] Фильтр по режиму работы работает
- [x] Частичное совпадение города (например, "Санкт" найдёт "Санкт-Петербург")

---

### 2. GET /api/v1/vacancies/{id} — детали вакансии

**Параметры path:**
- `vacancy_id` — ID вакансии (integer)

**Пример запроса:**
```
GET http://localhost:8000/api/v1/vacancies/1
GET http://localhost:8000/api/v1/vacancies/99999
```

**Ожидаемый результат:**
- Статус `200 OK` — если вакансия найдена ✅
- Статус `404 Not Found` — если вакансия не существует ✅

**Что проверить:**
- [x] Запросить существующий ID (например, 1)
- [x] Запросить несуществующий ID (например, 99999)

---

### 3. POST /api/v1/vacancies/ — создание вакансии

**Тело запроса (JSON):**
```json
{
  "external_id": 99999,
  "title": "Тестовая вакансия",
  "timetable_mode_name": "Полный день",
  "tag_name": "IT",
  "city_name": "Москва",
  "published_at": "2026-03-10T12:00:00Z",
  "is_remote_available": true,
  "is_hot": false
}
```

**Ожидаемый результат:**
- Статус `201 Created` — новая вакансия ✅
- Статус `200 OK` — если `external_id` уже существует (дубликат) ✅
- Статус `422 Unprocessable Entity` — некорректные данные ✅

**Что проверить:**
- [x] Создать вакансию с уникальным `external_id`
- [x] Создать вакансию с существующим `external_id` (дубликат)
- [-] Отправить пустую строку в `title` (должен быть 422)
- [x] Отправить неверный формат даты (должен быть 422)
- [x] Отправить `is_hot: "да"` вместо boolean (должен быть 422)
- [x] Проверить ответ дубликата: `{"detail": "Vacancy with external_id already exists"}`

---

### 4. PUT /api/v1/vacancies/{id} — обновление вакансии

**Параметры path:**
- `vacancy_id` — ID вакансии

**Тело запроса (JSON) — полное обновление:** ✅
```json
{
  "title": "Обновлённая вакансия",
  "timetable_mode_name": "Удалённая работа",
  "tag_name": "Marketing",
  "city_name": "Санкт-Петербург",
  "published_at": "2026-03-10T12:00:00Z",
  "is_remote_available": true,
  "is_hot": true
}
```

**Тело запроса (JSON) — частичное обновление:** ✅
```json
{
  "title": "Только заголовок"
}
```

**Ожидаемый результат:**
- Статус `200 OK` — вакансия обновлена ✅
- Статус `404 Not Found` — вакансия не найдена ✅
- Статус `422 Unprocessable Entity` — некорректные данные ✅

**Что проверить:**
- [x] Обновить все поля вакансии
- [x] Частичное обновление (только `title`) — остальные поля не меняются
- [x] Обновить несуществующую вакансию (404)
- [?] Отправить некорректные данные (422)
- [x] `external_id` игнорируется при обновлении (не вызывает 500 ошибку)

---

### 5. DELETE /api/v1/vacancies/{id} — удаление вакансии

**Параметры path:**
- `vacancy_id` — ID вакансии

**Ожидаемый результат:**
- Статус `204 No Content` — вакансия удалена (тело ответа пустое)
- Статус `404 Not Found` — вакансия не найдена

**Что проверить:**
- [x] Удалить существующую вакансию ✅
- [x] Удалить несуществующую вакансию (404) ✅
- [x] После удаления проверить GET /api/v1/vacancies/{id} — должен вернуть 404 ✅

---

### 6. POST /api/v1/parse/ — ручной запуск парсинга

**Тело запроса:** нет

**Ожидаемый результат:**
- Статус `200 OK`
- Ответ: `{"created": <число новых вакансий>}`

**Что проверить:**
- [x] Первый запуск — создаёт вакансии (например, 28)
- [x] Повторный запуск — возвращает 0 (вакансии уже загружены)

## Возможные проблемы (баги)

Искать следующие аномалии:
1. **Неправильные HTTP-статусы** (например, 200 вместо 201 или 409)
2. **Валидация данных** (принимает ли некорректные данные)
3. **Дубликаты** (создаются ли дубликаты вакансий)
4. **Фильтры** (работают ли `city` и `timetable_mode_name` регистронезависимо)
5. **Серверные ошибки** (500 Internal Server Error)
6. **Частичное обновление** (затираются ли поля при обновлении)
7. **`external_id` при обновлении** (вызывает ли 500 ошибку)

## Примечания

- После каждого теста на удаление проверять, что вакансия действительно удалена (GET должен вернуть 404)
- При тестировании частичного обновления проверять, что неуказанные поля не изменились
- Фильтр по городу должен работать регистронезависимо (Москва, москва, МОСКВА)
