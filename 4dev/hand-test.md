# Инструкция для ручного тестирования API

**Дата:** 10 марта 2026  
**Приложение:** Selectel Vacancies API  
**Swagger UI:** http://localhost:8000/docs

---

## Эндпоинты для тестирования

### 1. GET /api/v1/vacancies/ — список вакансий

**Параметры query (опционально):**
- `timetable_mode_name` — фильтр по режиму работы
- `city` — фильтр по городу

**Пример запроса:**
```
GET http://localhost:8000/api/v1/vacancies/
GET http://localhost:8000/api/v1/vacancies/?city=Москва
GET http://localhost:8000/api/v1/vacancies/?timetable_mode_name=Full
```

**Ожидаемый результат:**
- Статус `200 OK` ✅
- JSON-массив с вакансиями (должно быть ~28 вакансий после парсинга) ✅
- У каждой вакансии: `id`, `title`, `city_name`, `published_at`, и т.д. ✅

---

### 2. GET /api/v1/vacancies/{id} — детали вакансии

**Параметры path:**
- `vacancy_id` — ID вакансии (integer)

**Пример запроса:**
```
GET http://localhost:8000/api/v1/vacancies/1
```

**Ожидаемый результат:**
- Статус `200 OK` — если вакансия найдена ✅
- Статус `404 Not Found` — если вакансия не существует ✅

**Что проверить:**
- Запросить существующий ID (например, 1)
- Запросить несуществующий ID (например, 99999)

---

### 3. POST /api/v1/vacancies/ — создание вакансии

**Тело запроса (JSON):**
```json
{
  "external_id": 99999,
  "title": "Тестовая вакансия",
  "timetable_mode_name": "Полный день",
  "tag_name": "IT",
  "city_name": "Москва",
  "published_at": "2026-03-10T12:00:00Z",
  "is_remote_available": true,
  "is_hot": false
}
```

**Ожидаемый результат:**
- Статус `201 Created` — новая вакансия
- Статус `200 OK` — если `external_id` уже существует (дубликат)

**Что проверить:**
- Создать вакансию с уникальным `external_id` ✅
- Создать вакансию с существующим `external_id` (должен вернуться 200) ✅

---

### 4. PUT /api/v1/vacancies/{id} — обновление вакансии

**Параметры path:**
- `vacancy_id` — ID вакансии

**Тело запроса (JSON):**
```json
{
  "external_id": 99999,
  "title": "Обновлённая вакансия",
  "timetable_mode_name": "Удалённая работа",
  "tag_name": "Marketing",
  "city_name": "Санкт-Петербург",
  "published_at": "2026-03-10T12:00:00Z",
  "is_remote_available": true,
  "is_hot": true
}
```

**Ожидаемый результат:**
- Статус `200 OK` — вакансия обновлена
- Статус `404 Not Found` — вакансия не найдена

**Что проверить:**
- Обновить существующую вакансию ❌️ `Internal Server Error` если обновлять `exernal_id`
- Обновить существующую вакансию без `exernal_id` = `192.168.65.1:63423 - "PUT /api/v1/vacancies/3 HTTP/1.1" 200 OK`
- Попробовать обновить несуществующую (404) ✅

---

### 5. DELETE /api/v1/vacancies/{id} — удаление вакансии

**Параметры path:**
- `vacancy_id` — ID вакансии

**Ожидаемый результат:**
- Статус `204 No Content` — вакансия удалена
- Статус `404 Not Found` — вакансия не найдена

**Что проверить:**
- Удалить созданную тестовую вакансию ❌️ `404 Not Found`
- Удалить старую вакансию ✅ `204 No Content`
- Попробовать удалить несуществующую (404) ✅

---

### 6. POST /api/v1/parse/ — ручной запуск парсинга

**Тело запроса:** нет

**Ожидаемый результат:**
- Статус `200 OK` ✅
- Ответ: `{"created": <число новых вакансий>}` = 1 ✅

**Что проверить:**
- Запустить парсинг повторно (должно вернуть 0, так как вакансии уже загружены) ✅

## Возможные проблемы (баги)

Искать следующие аномалии:
1. **Неправильные HTTP-статусы** (например, 200 вместо 201)
2. **Валидация данных** (принимает ли некорректные данные)
3. **Дубликаты** (создаются ли дубликаты вакансий)
4. **Фильтры** (работают ли `city` и `timetable_mode_name`)
5. **Серверные ошибки** (500 Internal Server Error)
