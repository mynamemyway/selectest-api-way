## Шаг 0: Анализ репы

- **Что сделал:** Изучил структуру проекта и файл `app/core/config.py` для определения переменных окружения.
- **Проблема:** В репозитории отсутствовал файл `.env`, необходимый для запуска приложения. Переменные окружения описаны в README.md, но их значения нужно было определить из кода.
- **Решение:** 
  1. Создал файл `.env` со следующими переменными:
     - `DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/postgres`

     - `PARSE_SCHEDULE_MINUTES=5`
  2. Почистил `.gitignore` от избыточных строк (шаблон GitHub), оставил минимальный набор для Python-тестового.

## Шаг 1: Исправление бага №1 - лишняя строка в `requirements.txt`

- **Что сделал:** Перед запуском приложения проверил зависимости в `requirements.txt`.
- **Проблема:** Дублирующая строка со странной версией FastAPI: `fastapi==999.0.0; python_version < "3.8"`.
- **Решение:** Удалил строку из `requirements.txt`.
- **Причина ошибки:** Вероятно добавленная несуществующая версия пакета (999.0.0).
- **Итог:**
    1. Создал и активировал `venv`
    2. Установил зависимости из `requirements.txt`

## Шаг 2: Исправление бага №2 - именование переменной db

- **Что сделал:** Запустил докер, получил лог с ошибкой
- **Описание проблемы:** Приложение не может корректно прочитать переменную окружения `DATABASE_URL` из файла `.env`.
- **Файл и строка:** `app/core/config.py:13`
- **Код до исправления:**
  ```python
  validation_alias="DATABSE_URL",
  ```
- **Код после исправления:**
  ```python
  validation_alias="DATABASE_URL",
  ```
- **Причина ошибки:** Опечатка в названии алиаса (`DATABSE_URL` вместо `DATABASE_URL`), из-за чего pydantic-settings не находит переменную окружения.
